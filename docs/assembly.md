# 组装指南

本文档介绍 InkSight 的硬件连接与上电检查流程，目标是让你在 10-20 分钟内完成可用硬件组装。

## 1. 组装前准备

- 主控板：ESP32-C3 SuperMini（推荐）
- 屏幕：4.2寸 SPI E-Paper
- 连接线：杜邦线 6 根以上
- 供电：USB-C（调试期建议）或锂电池方案（推荐 `505060-2000mAh` + TP5000，用于长期使用）

## 2. 4.2寸 + ESP32-C3 Pro mini / Super mini开发板 推荐接线

如果你使用推荐方案（`epd_42_c3`），建议按下表接线：

| ESP32-C3 | 墨水屏 |
|----------|--------|
| `GPIO4` | `CLK / SCK` |
| `GPIO6` | `DIN / MOSI` |
| `GPIO7` | `CS` |
| `GPIO1` | `DC` |
| `GPIO2` | `RST` |
| `GPIO10` | `BUSY` |
| `3V3` | `VCC` |
| `GND` | `GND` |

## 3. 供电建议

### 开发与排障

建议优先使用 **USB 直供**：

- 接线更简单
- 排障更容易
- 可以稳定使用串口日志

### 锂电池版本

如果你要做长期摆放版本，可以考虑：

- **单节锂电池（如聚合物软包）**
- **TP5000** 充电模块

当前推荐的锂电池型号是：

- **软包 `505060-2000mAh`**（505060为尺寸：5mm厚，50x60mm，容量2000mAh）

注意事项：

- 普通锂电池标称 3.7V（满电 4.2V），**必须接在开发板的 5V 引脚**，利用板载 LDO 降压到 3.3V，切勿直连 3.3V 引脚以免烧毁设备。
- TP5000 模块默认即为 4.2V 充电，无需短接任何焊盘。
- 实际续航与刷新频率、Wi-Fi 质量、运行模式密切相关

目前项目对外通常描述为“低频刷新场景下可做到月级续航，理想情况下可到数月”，不要把所有使用场景都理解成固定 6 个月。

## 4. 组装顺序建议

建议按以下顺序连接，便于定位故障：

1. 先接电源（VCC/GND）
2. 再接 SPI 主信号（CLK、DIN、CS）
3. 最后接控制线（DC、RST、BUSY）

这样即使出现黑屏或刷新异常，也能更快判断是供电问题还是信号线问题。

## 3. 首次上电检查

首次上电建议仅使用 USB 供电，不要同时接锂电池。检查项：

- ESP32-C3 能被电脑识别串口
- 屏幕上电后有初始化闪屏（或至少有刷新动作）
- 串口日志无持续报错（如初始化失败、内存错误）

## 6. 常见硬件问题

### 屏幕不显示

- 检查 `VCC / GND` 是否共地
- 检查 `RST / BUSY` 是否接反
- 检查 SPI 接线是否与当前构建环境一致
- 检查屏幕型号是否为 SPI 版本，且与你选择的驱动/尺寸匹配

### 刷新异常（花屏/局部不刷）

- 优先检查 `CLK / DIN / CS` 稳定性
- 缩短连接线长度
- 排查供电电压是否过低或不稳

### 启动反复重启

- 常见于供电不足或锂电池模块输出不稳
- 调试阶段请先改用 USB 直供排查

### Wi-Fi 正常但刷新慢

电子墨水屏刷新速度本身就比 LCD 慢，另外还会受以下因素影响：

- Wi-Fi 信号强度
- 当前模式是否需要外部数据或大模型
- 是否命中缓存

这通常不是“屏幕坏了”，而是系统整体链路时延的表现。

### Busy Timeout 排查

如果串口中出现 `Busy Timeout!`，表示驱动在等待墨水屏 **BUSY** 引脚状态变化时超时。

优先排查：

1. **BUSY 没接或接触不良**
2. **BUSY 接错脚位**
3. **BUSY 被直接拉到 3.3V / GND**
4. **屏幕模组丝印与想象中的脚位顺序不一致**

对于 ESP32-C3 推荐方案，请重点确认：`GPIO10` ↔ 屏幕 `BUSY`。

## 7. 下一步

硬件组装并确认无误后，你的设备已经准备好了。接下来请按照以下顺序完成软件配置：

1. **烧录固件**：查看 [Web 在线刷机指南](flash)，通过浏览器将固件写入 ESP32。
2. **配置设备**：查看 [设备配置指南](config)，为设备配网并设置要显示的模式。
3. **设置 AI 算力**：查看 [API Key 配置](api-key)，填入你的大模型密钥以驱动 AI 内容生成。
