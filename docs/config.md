# Web 在线配置

Web 在线配置页用于管理设备、配置内容模式、调整个性化参数、设置 AI 模型，并查看设备运行状态。

## 1. 入口与使用方式

- 入口：[`设备配置`](/config)
- 需要先登录账号
- 登录后先进入设备列表，再选择要配置的设备
- 未绑定设备时，可在页面中手动绑定，或等待新设备上线后自动识别

## 2. 页面结构

配置页分为 4 个标签页：

- 模式
- 个性化
- AI 模型
- 状态

保存按钮为“保存到设备”，配置按设备维度生效。

## 3. 模式页

模式页用于控制设备显示内容与预览操作：

- 核心模式、更多模式、自定义模式的启用/停用
- 单模式预览与重新生成预览
- 收藏模式
- 模式级设置（按模式覆盖）
  - 可覆盖服务商、模型、城市
  - 可编辑模式自定义参数（如 COUNTDOWN 事件）
- 自定义模式能力
  - AI 生成模式
  - 模板创建模式
  - 预览、保存、删除自定义模式
- 预览图可直接“应用到墨水屏”

## 4. 个性化页

个性化页用于设置全局生成偏好：

- 城市（全局默认）
- 语言（中文/英文/中英混合）
- 内容语气（积极/中性/深沉/幽默）
- 人设风格（预设 + 自定义）
- 刷新策略（random / cycle / time_slot / smart）
- 刷新间隔（分钟）

## 5. AI 模型页

AI 模型页支持文本与图像两套独立配置：

- 文本服务商、文本模型、文本 API Key
- 图像服务商、图像模型、图像 API Key

说明：

- API Key 输入框留空表示不修改已保存值
- 设备级 Key 未配置时，后端回退到环境变量

## 6. 状态页

状态页用于查看设备运行与统计信息：

- 在线状态、运行状态、上次在线时间
- 总渲染次数、缓存命中率、电压、电量、信号、错误次数
- 模式使用频率
- 可手动刷新状态

## 7. 保存与生效

- 点击“保存到设备”后写入设备配置
- 设备在线时通常可较快生效
- 设备离线时会在下次上线后生效

## 8. 兼容说明

- 后端 `webconfig` 的 `/config`、`/dashboard` 页面后续计划下线
- 建议统一使用 WebApp 的 `/config` 作为配置入口
- 后端保留 `/preview` 用于渲染预览与调试
